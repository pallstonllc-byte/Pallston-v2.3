02 House style

The signature should read like correspondence from a serious professional-services firm: quiet, exact, and consistent across the whole company. The name carries the only emphasis; everything else is secondary.

Do

  • Keep the exact structure, order, sizes, and colors.
  • Use your real name, title, email, and direct phone.
  • Leave the firm identity to the logo and pallston.com.
  • Keep it to this one signature for new mail and replies.

Don't

  • Add quotes, banners, social icons, or marketing lines.
  • Add extra colors, bold, or a second logo.
  • Attach the logo — always link the hosted URL.
  • Insert blank lines to “space it out” (see §08).
  • Retype a link's visible text to "update" it — the address underneath won't change (see §05).

03 Confidentiality notice

Standard notice text (used in the signature above):

“This email and any attachments are confidential and may be legally privileged, and are intended solely for the named recipient(s). If you are not the intended recipient, please notify the sender and delete this message; any use, disclosure, copying, or distribution is prohibited.”
Legal note: This is a general, modern confidentiality notice — not legal advice. Have Pallston’s General Counsel approve the final wording for the jurisdictions you operate in. Counsel may want to add: an EU/UK data-protection line, a “Pallston cannot guarantee email is secure or error-free” sentence, or specific handling language for government / Controlled Unclassified Information (CUI). Once approved, this is the only notice everyone uses.

05 Changing contact details safely

This is the #1 cause of a signature quietly sending to the wrong address. Outlook's link editor stores the visible blue text and the actual mailto:/tel: address in two separate fields. If you double-click the email text in an existing signature and retype it, you have only changed what people see — the link underneath still points at the old address until you edit it too.

Recommended: never hand-edit a link — regenerate the block

Use Pallston-Signature-Generator.html in this folder. Type your name, title, email, and phone; it builds the whole signature fresh, with the visible text and the mailto:/tel: links generated from the same values, so they can never drift apart. Copy the result and paste it in as a brand-new signature (replace the old one entirely rather than editing inside it).

If you must edit an existing signature directly in Outlook

Update both fields, not just the text:

  1. Right-click the email (or phone) link in the signature editor → Edit Hyperlink (Windows) / Edit Link (Mac).
  2. Update “Text to display” to the new email/phone.
  3. Update “Address” to mailto:new.address@pallston.com (or tel:+1##########) — this is the field people forget.
  4. Click OK, then send yourself a test email and click the link to confirm it opens the correct address.

Same rule for the website link and the logo's link-through — the visible text/image and the destination URL are independent; changing one never changes the other.

06 Hosting the logo

Email clients fetch the logo from a URL every time the message is opened — it must be publicly reachable and stable.

  1. Upload pallston-logo-chip.png to a permanent public HTTPS location — e.g. https://pallston.com/email/pallston-logo-chip.png (website assets folder or a CDN).
  2. Confirm it loads in a private/incognito browser window with no login required (Gmail and Outlook fetch it anonymously).
  3. Keep the URL stable. To update the logo company-wide later, replace the file at the same URL rather than changing the path.
  4. Do not embed the image as base64 or attach it — Outlook and Gmail handle those poorly (broken or “attachment” icons). Always link the hosted URL.
  5. Serve over HTTPS only; mixed/HTTP images are blocked by many clients.

07 Install

Recommended: open this page in a browser, select the signature (from the name through the end of the notice), copy, and paste into your client’s signature editor. Because the signature has no background, nothing is pasted behind it. For central deployment, use the §04 HTML directly in your signature-management tool.

Outlook (Windows) — File → Options → Mail → Signatures → New → paste → set as default for New messages and Replies.

Outlook (Mac) — Outlook → Settings → Signatures → “+” → paste. (If new Outlook for Mac strips formatting, paste again after enabling HTML, or use the §04 code.)

Gmail — Settings → See all settings → General → Signature → Create new → paste → choose as default for new mail and replies.

Apple Mail — Mail → Settings → Signatures → “+” → uncheck “Always match my default message font” → paste.

08 Rendering & compatibility

  • No background. Every table and cell is transparent (background-color:transparent), so the signature never pastes a white or colored box — it sits directly on the email.
  • No extra line spacing. Each line is its own table row with padding:0 and an exact line-height (mso-line-height-rule:exactly). There are no <p>, <div>, or <br> tags, which is what normally causes Outlook to double-space a pasted signature.
  • Never add blank lines. If you want more air, adjust the cell padding in the code — don’t press Enter, which reintroduces the spacing problem.
  • Full-width confidentiality line. The outer table is width="100%" with no max-width, while the name/contact block is a separate inner table with no width set — so it stays compact and left-aligned, and only the notice stretches to use the compose window's actual width, wrapping onto fewer lines the wider the window is. It will run on one line if the window is wide enough for the full sentence; on a narrower compose pane it wraps like normal text.
  • Dark mode. The logo is on its own graphite chip, so it reads on any canvas with no swap. Text is dark by default; dark-mode clients (Outlook desktop, Gmail app, Apple Mail) invert it to light automatically. No media queries are used.
